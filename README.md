# AI Tutor тАФ Project 2: AI Full Stack Web App

A full-stack web app that lets a signed-up user pick a school subject, ask a question in plain English, and get an explanation back from Google's Gemini API.

## Features

- **Sign up** тАФ create an account with name, email, and password
  - Email must be unique
  - Password must be at least 8 characters
  - Password cannot be the same as the email or name
  - Password is hashed with `crypto.pbkdf2` (random salt per user) before being stored тАФ plaintext passwords are never saved
- **Sign in** тАФ authenticates against the stored hash and starts a session
- **Session-based access control** тАФ the tutor page is only reachable while signed in. Visiting `/tutor` without a session redirects to `/signin`
- **AI Tutor page** (main page, signup/signin-gated)
  - Pick a subject from a dropdown: Math, Science, History, English, Computer Science
  - Type a question
  - Submit to get an explanation generated by the Gemini API, displayed on the same page
- **Profile page** (signup/signin-gated) тАФ view your account, update your name and/or password (same validation rules as signup), or delete your account entirely
- **Log out** тАФ ends the session and returns to the sign-in page

## Tech Stack

- **Backend:** Node.js, Express
- **Views:** EJS
- **Database:** MongoDB (Atlas)
- **Auth/session:** `express-session`, `crypto` (password hashing)
- **AI model:** Google **Gemini API** (`gemini-2.5-flash`, REST `generateContent` endpoint)

## How to Run the Server

1. Clone/download the project and open the folder in a terminal.
2. Install dependencies:
   ```
   npm install
   ```
3. Create a `.env` file in the project root with:
   ```
   ATLAS_URI=your_mongodb_connection_string
   GEMINI_API_KEY=your_gemini_api_key
   SESSION_SECRET=any_random_string
   ```
4. Start the server:
   ```
   npm start
   ```
5. Open `http://localhost:3000` in a browser. You'll be redirected to the sign-up page first.

## API Used

**Google Gemini API** тАФ `gemini-2.5-flash` model, called via the REST `generateContent` endpoint (`https://generativelanguage.googleapis.com/v1beta/models/gemini-2.5-flash:generateContent`). The app sends the user's subject + question as a prompt and renders the model's text response on the tutor page. No SDK is used тАФ the call is made with the built-in `fetch`.

## Database Schema

**Database:** `cs355db`
**Collection:** `users`

| Field      | Type   | Description                                  |
|------------|--------|-----------------------------------------------|
| `name`     | String | User's display name                          |
| `email`    | String | Unique, used as the sign-in identifier        |
| `salt`     | String | Random hex salt generated per user            |
| `password` | String | `pbkdf2` hash of the password + salt (never plaintext) |

Example document:
```json
{
  "_id": "ObjectId(...)",
  "name": "Jane Doe",
  "email": "jane@example.com",
  "salt": "a1b2c3...",
  "password": "9f8e7d..."
}
```

### CRUD Operations

| Operation | Where | Description |
|-----------|-------|--------------|
| Create | `POST /users/signup/submit` | Inserts a new user document |
| Read | `POST /users/signin/submit`, `GET /profile` | Looks up a user by email at signin; fetches the current user's data for the profile page |
| Update | `POST /profile/update` | Updates the user's `name` and/or `password`/`salt` |
| Delete | `POST /profile/delete` | Deletes the user's document and destroys their session |

## Deployment

This app is deployed on [Render](https://render.com).

- **Live app:** _add your Render URL here_
- **GitHub repo:** _add your repo link here_

To deploy your own copy:
1. Push the repo to GitHub (`.env` is excluded via `.gitignore` тАФ never commit it).
2. On MongoDB Atlas, go to Network Access and allow access from `0.0.0.0/0` so Render's servers can connect.
3. On Render, create a new Web Service from the repo with:
   - Build command: `npm install`
   - Start command: `npm start`
4. Add environment variables in Render's dashboard: `ATLAS_URI`, `GEMINI_API_KEY`, `SESSION_SECRET`.
5. Deploy. Render assigns a public URL (`your-app-name.onrender.com`).

## Notes

- All signup/signin validation happens server-side in `routes/users.js`.
- Sessions expire after 2 hours of inactivity (configurable in `app.js`).
- The `.env` file is excluded from version control and must be created locally with your own MongoDB connection string and Gemini API key.
